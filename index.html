파이어베이스를
연결해주세요.

이 사이트가 학생들 사이에서 실시간으로 의견을 주고받으려면 Firebase 데이터베이스가 필요합니다. 아래 단계를 따라 한 번만 설정해두면 4차시 내내 사용할 수 있어요.

1

Firebase 프로젝트 만들기

console.firebase.google.com 에 구글 계정으로 접속해서 "프로젝트 추가"를 누릅니다. 프로젝트 이름은 알아보기 편한 것이면 됩니다 (예: moral-ai-class). Google Analytics는 켜지 않아도 됩니다.

2

Firestore 데이터베이스 만들기

왼쪽 메뉴 "빌드 → Firestore Database" → "데이터베이스 만들기" 클릭.

  • 위치: asia-northeast3 (서울) 추천
  • 규칙: "테스트 모드에서 시작" 선택 (30일 동안 자유롭게 사용 가능)
3

익명 로그인 켜기

왼쪽 메뉴 "빌드 → Authentication" → "시작하기" → "Sign-in method" 탭에서 "익명"을 선택해서 "사용 설정" 켜기. (학생들이 별도 로그인 없이 사용할 수 있게 해줍니다.)

4

웹 앱 등록하고 설정값 복사

왼쪽 메뉴 "프로젝트 개요" 옆 톱니바퀴 → "프로젝트 설정". 스크롤 내려서 "내 앱" 영역에서 </> (웹) 아이콘 클릭. 별명 입력하고 "앱 등록". 표시되는 firebaseConfig 객체를 통째로 복사하세요.

5

HTML 파일에 붙여넣기

이 HTML 파일을 메모장이나 코드 편집기로 열고, 아래 부분을 찾아 복사한 값으로 교체합니다.

const FIREBASE_CONFIG = {
  apiKey: "여기에-붙여넣으세요",
  authDomain: "여기에-붙여넣으세요",
  projectId: "여기에-붙여넣으세요",
  ...
};
6

파일 호스팅하기

학생들이 접속하려면 인터넷에 올려야 합니다. 가장 빠른 방법:

  • Netlify Drop — HTML 파일을 페이지에 드래그만 하면 URL이 즉시 생깁니다 (회원가입 불필요, 24시간 무료)
  • 또는 GitHub Pages, Firebase Hosting, Vercel 등
주의: file://로 직접 열면 Firebase가 동작하지 않습니다. 반드시 https://로 시작하는 주소로 접속해야 해요.
7

학생들에게 안내

호스팅된 URL을 학생들에게 알려주세요. 처음 접속하면 수업 코드(예: 6-2-2026)와 이름을 입력합니다. 같은 수업 코드를 입력한 학생들끼리만 의견이 공유돼요. 여러 반에서 쓸 때 반마다 다른 코드를 주면 데이터가 섞이지 않습니다.

＋

(선택) 좀 더 안전하게 — 보안 규칙 변경

테스트 모드는 30일 후 만료됩니다. 그 전에 Firestore의 "규칙" 탭에서 아래로 바꾸면 무기한 사용 가능하면서, 익명 로그인된 사용자만 데이터에 접근하게 제한됩니다:

rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /classes/{classCode}/{document=**} {
      allow read, write: if request.auth != null;
    }
  }
}
